燕山大学 《面向对象程序设计》课程三级项目 项目研究报告 人员信息管理系统 项目名称：   人员信息管理系统 学    院：   人工智能学院（软件学院） 专业班级：   软件工程 姓    名：   李沂航 指导教师：   郜山权 完成日期：   2026年5月 摘  要 本项目设计并实现了一个基于C++面向对象编程技术的人员信息管理系统。系统采用链表作为核心数据结构，通过类继承体系管理学生、教师和后勤人员三类人员的信息。系统实现了人员信息的添加、查询、修改、删除、统计、排序等基本功能，并利用链表派生出 栈 、队列和集合三种数据结构，分别演示后进先出、先进先出和集合运算的特性。查询功能支持三级模糊匹配（精确子串、同义词扩展、编辑距离容错），并集成了基于 Ollama 本地大语言模型的AI智能查询功能。系统还支持数据的持久化存储以及MySQL/CSV格式的导入导出。通过本项目，深入实践了C++面向对象编程的核心概念，包括类继承、多态、运算符重载、 析构函数 管理等关键技术。 关键词： 面向对象编程；链表；类继承；模糊查询；数据结构 1  前言 面向对象程序设计是现代软件开发的核心理念之一，通过封装、继承和多态等机制，可以构建出高内聚、低耦合的软件系统。本课程三级项目要求设计实现一个人员信息管理系统，旨在通过实际项目开发，深入理解和运用C++面向对象编程技术。 本项目的开发目标包括：（1）使用链表实现人员信息的动态管理；（2）通过类继承体系实现多态的人员类型管理；（3）从链表类派生出 栈 、队列和集合，演示数据结构的基本操作；（4）实现模糊查询和AI智能查询等高级功能；（5）支持数据的持久化和外部格式导入导出。 项目采用C++17标准开发，编译环境为MinGW g++，终端界面使用ANSI转义序列进行美化。全部代码采用header-only方式组织，无外部依赖。 2   项目需求分析 2.1  问题描述 开发一个人员信息管理系统，用于管理学生、教师及后勤人员的相关信息，具备查询、统计、添加、删除、修改等功能。系统需要使用链表实现，链表元素中要有指针指向动态分配的内存空间， 练习析构函数 的操作规则。 2.2  功能需求 人员信息的增删查改：支持学生、教师、后勤人员三种类型的记录管理 链表类设计：至少包含Node类和List类，Node 类负责 结点本身的初始化和空间回收，List 类负责 整个链表的管理工作 派生类设计：从List类派生出Stack和Queue类，使其具有自身的操作特性（后进先出/先进先出） 集合运算：从List类派生出Set类，利用“+”和“-”运算符重载实现集合合并和集合相减操作 模糊查询：支持按姓名、学院等字段进行模糊搜索 数据持久化：支持将数据保存到文件及从文件加载 数据导入导出：支持MySQL和CSV格式的数据导入导出 2.3  非功能需求 代码遵循C++17标准，零外部依赖 终端界面美观、操作友好 学号格式校验：12位纯数字，含年份、学院代码、专业代码 操作日志：记录所有增删改操作，最多保留100条 3  总体设计 3.1  类继承体系 系统采用三层类继承体系设计。Person为 抽象基类 ，定义了人员共有的属性和接口；Student、Teacher、Staff三个派生类分别实现各类人员的特有属性和显示方法。 类名 继承关系 特有属性 Person 抽象基类 编号、姓名、年龄、性别 Student Person → Student 学院、专业、年级 Teacher Person → Teacher 院系、职称 Staff Person → Staff 部门、岗位 图1  类继承体系 3.2  数据结构设计 系统使用三种从List类派生的数据结构： （1） 栈 （Stack）：从List类派生，实现push、pop、top等操作，演示后进先出（LIFO）特性。用户可将人员信息压 栈 ，从 栈 顶弹出。 （2）队列（Queue）：从List类派生，实现enqueue、dequeue、front等操作，演示先进先出（FIFO）特性。用户可将人员排入队列，从队首离队。 （3）集合（Set）：从List类派生，保证元素互不重复。通过重载“+”运算符实现集合并集，重载“-”运算符实现集合差集。 3.3  主菜单模块 系统主菜单包含9个功能模块：添加记录、查询记录、修改记录、统计信息、排序浏览、数据结构演示、数据导入导出、操作日志、退出系统。其中查询记录子菜单包含高级查询、显示所有、AI智能查询三个选项，查询结果展示后可直接进行删除或修改操作。 4  各功能模块的设计和开发 4.1  添加记录模块 添加记录支持三种人员类型：学生、教师和后勤人员。学生添加采用三级联动选择机制，从燕山大学实际学院专业数据中选择，支持3位快捷码直接定位。学生支持单个添加和批量添加两种模式。学号格式严格校验：12位纯数字，前4位为年份，第5-6位为学院代码，第7-8位为专业代码，后4位为序号。 4.2  查询记录模块 查询功能采用两步式设计：第1步选择要显示的字段（投影），第2步选择筛选条件（选择）。支持三级模糊匹配：第一级为 精确子串匹配 ；第二级为同义词扩展；第三级为基于 Levenshtein 编辑距离的容错匹配。 查询结果展示后，系统提供删除和修改两种后续操作，用户可直接从查询结果中选择记录进行操作，无需返回主菜 单重新 查找。 4.3  修改记录模块 修改功能支持按编号查找后逐字段修改。对于每种人员类型，系统列出所有可修改的字段及当前值，用户选择字段后输入新值。修改操作自动记录到操作日志中，包含修改前后的值对比。 4.4  删除记录模块 删除功能集成在查询结果展示之后。用户查询到目标记录后，可直接选择删除。系统采用二次确认机制： 先展示待 删除记录的完整信息，用户确认后才执行删除。删除操作同样记录到操作日志。 4.5  统计信息模块 统计模块显示总人数及三类人员的分布情况，并以柱状图形式展示各学院学生分布、各院系教师分布和各部门后勤分布，直观反映人员构成。 4.6  排序浏览模块 排序模块将所有人员收集到std::vector中，使用std::sort配合lambda比较器实现多种排序方式：按编号升序、按姓名升序、按年龄升序、按类型排序。排序结果直接展示，操作记录到日志。 4.7  数据结构演示模块 4.7.1   栈 演示 栈 从List类派生，实现后进先出操作。用户可将人员信息压 栈 （push），从 栈 顶弹出（pop），查看 栈 顶元素（top），以及显示 栈 内容。 栈 的派生验证了继承机制的正确性。 4.7.2  队列演示 队列从List类派生，实现先进先出操作。用户可将人员排入队尾（enqueue），从队首出队（dequeue），查看队首元素（front），以及显示队列内容。 4.7.3  集合运算演示 集合从List类派生，保证元素不重复。通过运算符重载实现：“+”运算符计算集合并集（A+B），“-”运算符计算集合差集（A-B）。集合运算演示了运算符重载和集合数学概念的结合。 4.8  数据导入导出模块 数据导入导出支持两种格式：MySQL和CSV。导出至MySQL时生成完整的SQL脚本文件，包含建库、 建表和 插入语句；从MySQL导入时提供操作指引。CSV格式支持BOM头（UTF-8）以确保Excel正确识别中文，导入时自动识别学生、教师、后勤三种数据格式。 4.9  AI智能查询模块 AI智能查询模块通过调用本地 Ollama 大语言模型服务，实现自然语言查询。工作流程：（1）通过/ api /tags接口自动检测可用模型；（2）用户选择模型后，将系统中所有人员数据摘要和用户问题组合成prompt；（3）通过/ api /generate接口发送请求，采用非流式模式；（4）解析响应并展示AI回答。该模块的请求体通过临时文件传递，避免命令行长度限制。 4.10  操作日志模块 操作日志使用std::deque< LogEntry >存储，最多保留100条记录。每条日志包含时间戳、操作类型和操作详情。添加、修改、删除、排序操作均自动记录到日志。日志按时间倒序显示，操作类型以彩色标签区分。 5  结论 本项目成功实现了一个功能完善的人员信息管理系统，涵盖了面向对象编程的核心技术要点。通过本项目的开发，深入理解并实践了以下关键技术： 类继承与多态：通过Person 抽象基类 和Student/Teacher/Staff派生类的层次设计，实现了统一接口下的差异化行为 析构函数 管理：链表Node类和List类 的析构函数 正确处理了动态内存的递归释放，避免了内存泄漏 运算符重载：Set类中“+”和“-”运算符的重载，使集合运算的表达更加直观自然 泛型编程：std::sort配合lambda比较器实现多字段排序，展示了C++标准库算法的灵活运用 模糊查询：三级模糊匹配机制大大提升了查询的智能性和用户体验 AI集成：通过 Ollama 本地模型实现自然语言查询，探索了传统软件与AI能力的结合方式 未来改进方向包括：引入图形用户界面（GUI）、实现网络化多人协作、增加权限管理机制等。本项目为后续学习和开发奠定了坚实的基础。 6  主要参考文献 [1] Stanley B.
