## 二、赛题核心要求

### 技术流程
```
图像预处理 → 特征提取 → 模型训练 → 推理优化 → 提交评分
```

### 评分排名指标（按优先级）

| 优先级 | 指标 | 说明 |
|--------|------|------|
| 🥇 **第一** | **F1分数（宏平均）** | 主要排名依据，越高越好 |
| 🥈 **第二** | **模型推理时间** | F1相同时，推理时间越短越好 |
| 🥉 **第三** | **代码执行效率** | 前两项相同时，代码效率越高越好 |

### 核心挑战
> **在有限计算资源下，精准平衡 模型分类精度（F1）与 推理速度**

这不是单纯的"刷分"比赛，而是**工程优化**比赛。

## 六、竞赛策略总结

### 制胜关键
1. **F1优先**：先保证分类精度，再优化速度
2. **模型选型**：用 EfficientNet-B0/B1 或轻量级 ResNet 作为主力模型
3. **数据增强**：利用Albumentations做针对性增强
4. **剪枝+量化**：训练完成后做剪枝和量化，大幅减少推理时间
5. **ONNX+TensorRT**：导出ONNX后用TensorRT加速推理
6. **Batch调优**：找到最优batch size平衡速度与精度

### 用六步框架看竞赛策略

| 六步 | 对应竞赛问题 | 答案 |
|------|-------------|------|
| **建立地图** | 比赛技术栈全貌 | Python→PyTorch→CNN→调优→优化→提交 |
| **统一抽象** | 核心评分公式 | **最终得分 ∝ F1_macro / 推理时间** |
| **核心矛盾** | 最关键的 trade-off | **精度 vs 速度**——F1 优先，同 F1 比速度 |
| **因果链** | 怎么涨 F1？ | 数据增强→模型选型→超参数→蒸馏→后处理 |
| **压缩** | 三句话说清策略 | 先涨 F1（增强+选型）→ 再稳 F1（调参+正则）→ 最后压速度（剪枝+量化+ONNX） |
| **检索检查** | 比赛当天能否快速决策 | 准备 3 套方案，按平台情况选 |
